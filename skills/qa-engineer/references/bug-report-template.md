## Bug Severity Definitions

| Severity | Definition | Example |
|----------|-----------|---------|
| CRITICAL | System broken, data loss, security breach, no workaround | Auth bypass, data corruption, crash on startup, app won't start |
| HIGH | Major feature broken, poor workaround | Payment fails for certain card types, form submits but doesn't save |
| MEDIUM | Feature works but not as specified, acceptable workaround | Validation missing on one field, wrong error message |
| LOW | Cosmetic, minor inconvenience | Typo in UI, slight misalignment, console warning |
